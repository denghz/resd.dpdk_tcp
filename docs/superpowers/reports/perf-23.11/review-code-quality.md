## Strengths

- **Feature gating is rigorous.** `bench-internals` is additive and off by default. Production `cargo build` compiles `tcp_timer_wheel` as `pub(crate)` (compiled output byte-identical to pre-change, per commit `cd3cf20`); `EngineNoEalHarness` and its test_support module compile away entirely. The cfg-dual `mod tcp_timer_wheel` declaration in `lib.rs` (lines 45-48) is a clean way to flip outer module visibility without touching the file's internal `pub` modifiers.
- **H1 implementation faithfully mirrors `Engine::poll_once`.** Compare engine.rs:2079-2090 (real path, `RefCell<SmallVec<[ConnHandle; 8]>>` with `clear() + extend(ft.iter_handles()) + drain(..)`) against engine.rs:144-152 (harness, `Vec<ConnHandle>` with `clear() + extend(...) + for &h in &...`). The difference (`drain` vs reference iteration) is justified — the harness has no second use of the scratch and avoids draining only because it does not mutate-while-borrow with a `RefCell`. Behavior is equivalent for the bench's purpose.
- **Pre-allocation lives at `new()` time, not first-poll time.** `Vec::with_capacity(flow_capacity)` runs in the constructor; bench setup (`EngineNoEalHarness::new(64, ...)`) sizes the scratch to 64 slots — well above the 256 timers `pre_populate_timers` adds (those go in the wheel, not the scratch) and adequate for any realistic bench flow count.
- **Comment hygiene is high.** Hot rationale (H1 commit `5b8ee71`, the `EngineNoEalHarness` doc, the `let _ = timer_tick_ns` placeholder, the BATCH=128 / XOR-fold tsc_read methodology) is explained where non-obvious. The `csv_row.rs` doc-comment explicitly captures the 35→40 column transition with a concrete breakdown.
- **Backwards-compat for the CSV schema is tested.** `csv_row_deserialize_tolerates_missing_task_2_8_columns` (in `tools/bench-common/tests/csv_row_roundtrip.rs`) constructs a 35-column legacy header via the csv crate (correctly handling the JSON-with-commas in `dimensions_json`) and round-trips it through the 40-column deserializer. The `unwrap_or(None)` collapse at deserialize time is the right shape.
- **All five `EngineNoEalHarness` integration tests pass cleanly** (`cargo test -p dpdk-net-core --test engine_no_eal_harness --features bench-internals` → `5 passed; 0 failed`). The full workspace (`cargo check --workspace`) and `cargo test -p bench-common` (12 lib + 8 csv-roundtrip tests) also pass, confirming the 11 mechanical CSV-row construction edits compile and round-trip.
- **CSV schema extension is mechanically uniform across 11 affected crates.** Each tool's `CsvRow { ... }` construction received identical `cpu_family: None, cpu_model_name: None, dpdk_version_pkgconfig: None, worktree_branch: None, uprof_session_id: None,` block. Tools that emit non-bench-micro rows (bench-ab-runner, bench-e2e, bench-offload-ab tests, bench-report writers + tests, bench-stress, bench-vs-linux RTT + wire-diff, bench-vs-mtcp burst + maxtp) all include a brief comment explaining "blank for non-bench-micro tools." Only `bench-micro/src/bin/summarize.rs` populates them.
- **Methodology change is documented and the negative result is preserved.** The H1 (iter_custom batching) on `tsc_read` is explicitly retained even though it didn't move the median, with the explanation that "this proved criterion overhead was NOT the limiter." That kind of negative-result preservation is valuable for future readers.

## Notes

- **No data-race patterns introduced.** The harness owns its `FlowTable`, `TimerWheel`, `EventQueue`, `now_ns`, `conn_handles_scratch` directly (no `RefCell`, `Cell`, `Arc`, or atomic state added). All methods take `&mut self` (except `tuple_lookup` and `now_ns` which are read-only). Single-threaded by construction.
- **No `unwrap()` introduced in non-test code.** `summarize.rs` uses `.ok()?` on `Command::output()` which propagates None correctly. Tests use `.unwrap()` consistently as is standard for test code.
- **No silent fallback paths.** Every `Option` return is either flagged in doc-comments as "missing/absent" or surfaced as a `None` cell at the CSV layer. No dropped errors.
- **No dead code.** All imports in `test_support` (`ConnHandle`, `FlowTable`, `FourTuple`, `TcpConn`, `EventQueue`, `TimerId`, `TimerKind`, `TimerNode`, `TimerWheel`) are used. `HARNESS_OWNER` is used. The only "discarded" symbol is `timer_tick_ns` (finding #1).
- **No hot-path counters introduced.** Per `feedback_counter_policy`, only slow-path counters are default-on. The harness adds zero counter sites; the bench harness mutates exactly the same data structures the production engine touches, with no observability tax.
- **Hot-path regression check (poll family):** the H1 fix (commit `5b8ee71`) reduces `bench_poll_empty` from 1032.6 ns → 45.8 ns and `bench_poll_idle_with_timers` from 1131.5 ns → 50.7 ns — both well under §11.2's 100 ns upper. The fix is a strict improvement (no regression introduced).
- **Hot-path regression check (tsc_read family):** commit `2cc6829` (iter_custom batching + XOR-fold) is methodology-only at the median level (within noise on `bench_tsc_read_ffi`, -1.69% on `bench_tsc_read_inline`). The negative result is preserved with rationale; no regression.
- **CSV schema invariance guard updated.** `tools/bench-common/src/csv_row.rs:519` (`columns_len_is_expected`) now asserts `COLUMNS.len() == 40` (was 35). The breakdown comment matches: 13 + 14 + 8 + 5 = 40. If any future change adds/removes columns, this test will fire — good safety net.
