## 1. Purpose and scope

Phase A8.5 is a targeted follow-up to A8 that addresses three concerns
surfaced during A8 review:

1. **"Duplicate" doesn't mean "skip."** A8 deliberately skipped 6 tcpreq
   probes as duplicate-of-Layer-A. That categorization is a judgment call;
   running the tests is the proof. A8.5 ports the 6 skipped probes and
   runs them — either confirming they pass (redundant but documented), or
   surfacing real gaps where my Layer-A-equivalence claim was wrong.

2. **Mechanical skip reasons should be unblocked when cheap.** Two
   corpus-level skip buckets are removed by small, bounded work:
   TX option-order drift (~12 ligurio scripts wedged by our emission
   order), and Google packetdrill env-helper dependencies
   (`scripts/defaults.sh` / `set_sysctls.py` — ~160 scripts blocked at
   the shell-init step).

3. **`shutdown()` is a real POSIX API gap.** Our public FFI exposes
   `dpdk_net_close` but not `dpdk_net_shutdown`. ~11 ligurio scripts use
   `shutdown(fd, SHUT_*)`. A8.5 adds `dpdk_net_shutdown` to the
   production FFI (full-close semantics only; half-close returns
   `-EOPNOTSUPP`), unblocks the corpus scripts, and documents the
   POSIX-subset deviation as a new §6.4 row.

A8.5 also pins crash-safety for 6 connection-impacting corpus scripts
(ICMP ingress, malformed-syscall-arg handling, ICMP-driven PMTUD) —
scripts whose engine gaps could cause real connection breakage on bad
peer behavior.

### 1.1 In scope

Five parallel workstreams.

- **A — `tcpreq-redundant-port`**: port 6 tcpreq probes from the
  A8-skipped bucket to the `tools/tcpreq-runner/` crate:
  - `ZeroChecksumTest` — MUST-2/3 (checksum generation + validation).
    Probe injects SYN with zero checksum, asserts engine rejects
    (`tcp.rx_bad_csum` bumps; no handshake).
  - `OptionSupportTest` — MUST-4 (EOL / NOP / MSS options MUST be
    supported). Probe injects SYN with each option in isolation,
    asserts accepted + handshake completes.
  - `UnknownOptionTest` — MUST-6 (unknown options MUST be ignored).
    Probe injects SYN with a reserved option kind, asserts accepted +
    handshake completes.
  - `IllegalLengthOptionTest` — MUST-7 (illegal option length MUST be
    handled without crashing). Probe injects SYN with a runaway-length
    option; asserts `tcp.rx_bad_option` bumps OR RST emitted; no crash.
  - `MSSSupportTest` — MUST-14 (send + receive MSS option). Probe sends
    MSS in SYN, asserts our SYN-ACK carries MSS, and our outbound
    segments respect the effective send MSS.
  - `RstAckTest` — reset processing. Probe injects RST in various states
    (SYN_SENT, ESTABLISHED, close-path states); asserts expected
    state transitions + conn counters.

  Each probe lands as a `pub fn <name>() -> ProbeResult` in its module,
  and as a `#[test]` in `tools/tcpreq-runner/tests/probe_<module>.rs`.
  `run_all_probes()` in `lib.rs` extends from 4 → 10 calls.
  `SKIPPED.md` rows for these 6 move from "Skipped — duplicate Layer
  A/B coverage" to "Ported."

- **B — `tx-option-order-align`**: reorder TX option emission in
  `crates/dpdk-net-core/src/tcp_output.rs::build_segment` from
  `<mss, sack_permitted, timestamps, window_scale, nop>` to Linux
  default `<mss, nop, window_scale, sack_permitted, timestamps>`. Both
  orders are RFC-legal (RFC 9293 §3.2 is order-agnostic on received);
  the change is corpus-compatibility alignment, not a semantic fix.

  Unconditional change — no runtime knob (configurable TX order is
  premature polymorphism; no consumer depends on a specific order).
  Unit tests asserting byte-exact option-block output update to the
  new order (~3-5 sites).

  Unlocks ~12 ligurio `mss/*.pkt` scripts previously wedged on order
  drift.

- **C — `google-env-stubs`**: add `0007-google-env-stubs.patch` to the
  packetdrill-shim patch stack. The patch places two files inside
  `third_party/packetdrill/gtests/net/packetdrill/scripts/`:

  - `defaults.sh` — empty no-op stub. Scripts doing
    `source scripts/defaults.sh` get exit 0 without any sysctl /
    environment mutation.
  - `set_sysctls.py` — minimal Python file that no-ops on `source`
    (packetdrill sources Python files via its inline interpreter).
    If individual scripts read specific sysctl values, the stub returns
    conservative TCP defaults (`tcp_timestamps=1`, `tcp_sack=1`,
    `tcp_window_scaling=1`) rather than leaving them unset.

  Patch-overlay placement chosen over runtime fixture for reviewability
  (static tree, version-pinned, works identically in CI vs. local).

  After stubs land, re-run google corpus. Classify newly-runnable-or-now-
  failing-on-engine scripts. Pin `GOOGLE_RUNNABLE_COUNT` at the new
  value (estimated 5-15; most scripts still fail on engine gaps like CC,
  fast-retransmit, ECN — those remain in SKIPPED.md with updated
  reasons reflecting the new primary blocker).

- **D — `dpdk_net_shutdown`**: add POSIX `shutdown()`-subset public API.

  Signature (in `crates/dpdk-net/src/api.rs`, mirrored in `dpdk_net.h`
  via cbindgen):

  ```c
  #define DPDK_NET_SHUT_RD    0
  #define DPDK_NET_SHUT_WR    1
  #define DPDK_NET_SHUT_RDWR  2

  int32_t dpdk_net_shutdown(dpdk_net_conn_t h, int32_t how);
  ```

  Semantics:
  - `how == DPDK_NET_SHUT_RDWR` — behaves identically to
    `dpdk_net_close(h, 0)`. Transitions the connection through the
    standard FIN handshake to TIME_WAIT or CLOSED. Returns 0 on
    success, `-EINVAL` on unknown handle.
  - `how == DPDK_NET_SHUT_RD` or `how == DPDK_NET_SHUT_WR` — returns
    `-EOPNOTSUPP`. Connection state is unchanged.
  - Any other `how` value — returns `-EINVAL`.

  Implementation is a thin wrapper. No new engine code, no new
  connection state, no new counter. The test-FFI re-exposes the same
  function so packetdrill-shim can dispatch `shutdown(fd, how)` syscalls.

  Unblocks most of ~11 ligurio `shutdown/*.pkt` corpus scripts (the
  ones using SHUT_RDWR). Scripts using SHUT_RD or SHUT_WR remain
  skipped with updated reason ("half-close not implemented; spec §6.4
  AD-A8.5-shutdown-no-half-close").

- **G — `crash-safety-corpus`** (6 scripts): unblock and pin as
  runnable these connection-breakage-relevant corpus scripts:

  - `testcases/tcp/ICMP/icmp-all-types.pkt` — injects ICMP redirect,
    source-quench, destination-unreachable, echo-request. Engine MUST
    drop unexpected ICMP types without crashing the conn or the engine.
    `ip.rx_icmp` counter increments; no state change for non-frag-needed
    types.
  - `testcases/tcp/syscall_bad_arg/fastopen-invalid-buf-ptr.pkt` —
    FastOpen is `§6.3` row `NO`. Our API returns `-EOPNOTSUPP`; test
    asserts that return.
  - `testcases/tcp/syscall_bad_arg/sendmsg-empty-iov.pkt` —
    `dpdk_net_send(conn, buf, 0)` MUST return 0 (no-op). Existing API
    behavior; test pins it.
  - `testcases/tcp/syscall_bad_arg/syscall-invalid-buf-ptr.pkt` — null
    `buf` pointer MUST return `-EINVAL`. Existing API behavior; test
    pins it.
  - `testcases/tcp/mtu_probe/basic-v4.pkt` +
    `testcases/tcp/pmtu_discovery/pmtu-10pkt.pkt` — ICMP-frag-needed
    reduces MSS per RFC 1191. `ip.rx_icmp_frag_needed` + `ip.pmtud_updates`
    counters bump; subsequent segment uses reduced MSS.

  Each script's classifier entry moves from `skipped-engine-gap` (or
  similar) to `runnable` in `tools/packetdrill-shim/classify/ligurio.toml`.
  Any engine fixes required to close real crash-safety gaps land in the
  same workstream (minor; if a larger engine change is needed, that's a
  scope-escalation flag per workstream acceptance).

### 1.2 Out of scope

- CC / recovery feature implementation (early-retransmit, fast-recovery,
  fast-retransmit dupACK, limited-transmit, SACK shift, loss-undo, IW10,
  receiver-RTT, ECN, slow-start observability). These are S2-A phase
  scope per the roadmap.
- Half-close semantics (actual SHUT_RD / SHUT_WR state machine). The
  A8.5 `dpdk_net_shutdown` API explicitly rejects half-close with
  `-EOPNOTSUPP` per §1.1 D.
- GRO-coalesced peer segment tests. A9's proptest + cargo-fuzz targets
  already cover parser crash-safety; adding corpus scripts adds no new
  signal.
- Nagle fine-grained segmentation tests. We run Nagle off by default
  (§6.4). The tests exercise Linux-specific Nagle-coalesced send
  patterns we don't implement.
- `MSG_EOR` / message-boundary corpus tests. Application-layer semantics
  outside Stage 1 byte-stream API scope.
- epoll / splice / sendfile / ioctl-SIOCINQ syscall stubs. These are
  kernel socket-API features not in our FFI surface.
- New runtime config knobs. A8.5 adds no `EngineConfig` / `ConnectOpts`
  fields. The knob-coverage audit from A8 T17 continues to pin current
  state.

## 3. Design decisions

Recorded during brainstorming (2026-04-23 Q1–Q3).

### 3.1 Scope — tests to add

**Included (A+B+C+D+G-subset):**
- A: 6 tcpreq "redundant" probes (ported to verify the redundancy claim,
  not to add new signal).
- B: TX option-order alignment.
- C: Google defaults.sh + set_sysctls.py stubs.
- D: `dpdk_net_shutdown` public FFI.
- G subset (6 scripts): icmp-all-types, 3 syscall_bad_arg, 2 PMTU.

**Excluded with citations:**
- E (epoll / splice / sendfile stubs) — kernel-socket-API shape, not
  trading-relevant.
- F (CC / recovery features) — S2-A scope.
- G minus subset (GRO, Nagle, MSG_EOR) — A9 fuzz covers parser
  crash-safety; adding corpus scripts is redundant.

### 3.2 `dpdk_net_shutdown` — Option 2: EOPNOTSUPP for half-close

**Rejected:** Option 1 (API supports half-close semantically; tests only
cover full-close). Would introduce untested half-close state-machine
edges that could crash on an unlucky packet.

**Chosen:** Option 2. SHUT_RDWR works, SHUT_RD / SHUT_WR return
EOPNOTSUPP. Minimal API addition, no untested paths, clean future
expansion seam. Half-close is deferred to whenever a consumer actually
needs it (probably never — Stage 1 byte-stream API treats full-close
as the only close mode).

### 3.3 G subset — crash-safety scripts selected

**Criterion:** script unblocks a path where failure CAN crash the
engine or break a connection under unexpected peer behavior.

- **Included (6 scripts):** icmp-all-types, 3 syscall_bad_arg, 2 PMTU.
- **Excluded (remainder):** GRO / Nagle / MSG_EOR. A9's cargo-fuzz
  targets already cover "random bytes through the parser without
  crashing" — the corpus scripts would be repetitive signal.

### 3.4 B — unconditional TX order change (no knob)

RFC 9293 §3.2 is order-agnostic on both send and receive. Making TX
order configurable is premature polymorphism (no known consumer relies
on a specific order; adding a runtime switch adds test-surface + config
surface for zero benefit). Change unconditionally; the only cost is
updating ~5 unit tests that assert byte-exact frame output.

### 3.5 C — patch-overlay for env stubs

Alternative (runtime fixture in shim-runner) rejected for two reasons:
1. Reviewability: patch-overlay makes the stub content visible in the
   repo tree rather than hidden in test runner logic.
2. Consistency: patch-overlay works identically in CI vs. local shim
   builds; a runtime fixture introduces environment-dependent behavior.

## 4. Behavior changes

### 4.1 TX option-order alignment (B)

`tcp_output::build_segment`: for any segment carrying ≥2 TCP options,
emit options in Linux-default order: MSS → NOP (pad) → WSCALE → SACK-permitted
→ TIMESTAMPS.

- RFC 9293 §3.2 compliant (order-agnostic).
- No change in total segment size (options sum to the same byte count).
- No change in checksum correctness.
- Wire-visible to peers; any peer parsing our option bundle by position
  index now sees Linux-equivalent byte layout.

### 4.2 `dpdk_net_shutdown` addition (D)

New public FFI symbol exported from `libdpdk_net`. cbindgen-managed
header regeneration adds:
- `#define DPDK_NET_SHUT_RD 0`
- `#define DPDK_NET_SHUT_WR 1`
- `#define DPDK_NET_SHUT_RDWR 2`
- `int32_t dpdk_net_shutdown(dpdk_net_conn_t h, int32_t how);`

Implementation dispatches:
- `DPDK_NET_SHUT_RDWR` → internal call to the same path
  `dpdk_net_close(h, 0)` invokes.
- `DPDK_NET_SHUT_RD` or `DPDK_NET_SHUT_WR` → return `-EOPNOTSUPP` without
  touching engine state.
- Any other `how` → return `-EINVAL`.

ABI-stability: symbol is additive. Existing `dpdk_net_close` unchanged.

### 4.3 New §6.4 row: `AD-A8.5-shutdown-no-half-close`

POSIX `shutdown(2)` supports SHUT_RD / SHUT_WR for half-close
(simplex close of read or write direction). A8.5's `dpdk_net_shutdown`
rejects half-close with `-EOPNOTSUPP`. Rationale: half-close has edge
cases (retransmit on half-closed write side; RST timing on half-closed
read side) that Stage 1's byte-stream API doesn't need. Promotion gate:
any future phase (Stage 3+ HTTP/1.1 pipelining, WebSocket close-frame
handling) that needs half-close reopens.

## 5. Counters, knobs, events

**No new counters.** Corpus unlocks use existing counters
(`ip.rx_icmp_frag_needed`, `tcp.rx_bad_option`, `tcp.rx_bad_csum`,
`conn_close`, `conn_rst`, etc.).

**No new knobs.** A8 T17's static drift detector continues to pin the
current 38-field / 28-scenario inventory.

**No new events.** Shutdown emits existing `DPDK_NET_EVT_TCP_STATE_CHANGE`
and (on connection teardown) `DPDK_NET_EVT_ERROR` per existing close
semantics.

## 8. End-of-phase review gates

Two reviewers run in parallel (opus 4.7) as the final task, per durable
memory `feedback_phase_mtcp_review` + `feedback_phase_rfc_review`:

- **`mtcp-comparison-reviewer`**: evaluate the new public API surface
  (`dpdk_net_shutdown`) against mTCP's `mtcp_shutdown` for API shape
  alignment. Evaluate TX option-order alignment change against mTCP's
  TX emitter order (should converge). No expected new Accepted
  Divergences unless API shape differs from mTCP.
- **`rfc-compliance-reviewer`**: evaluate TX option-order change against
  RFC 9293 §3.2 (should be compliant — order-agnostic). Evaluate
  `dpdk_net_shutdown` EOPNOTSUPP-on-half-close against POSIX
  `shutdown(2)` semantics. Expected new Accepted Deviation:
  `AD-A8.5-shutdown-no-half-close`. Evaluate 6 tcpreq Layer-C probes
  against their RFC clauses (MUST-2/3, MUST-4, MUST-6, MUST-7, MUST-14,
  Reset) — all should be PASS.

Tag `a8.5-test-coverage-complete` is cut only when both reports are
clean (zero open `- [ ]`).
